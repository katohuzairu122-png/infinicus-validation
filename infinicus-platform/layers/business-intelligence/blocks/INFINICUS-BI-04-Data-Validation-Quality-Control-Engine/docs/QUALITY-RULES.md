# Supported Quality Rules

## required

Fails when the field is missing, null, or empty.

## type

Checks canonical semantic data type.

## range

Checks minimum and maximum values.

## pattern

Checks a regular-expression pattern.

## allowed_values

Checks whether the value belongs to an approved set.

## unique

Checks duplicate values inside the evaluated dataset.

## referential

Checks whether a value exists in a supplied reference set.

## timeliness

Checks whether a timestamp is within an approved age.
